## 1. 视频理解基础

### Q1：3D CNN 与 2D CNN + RNN 在视频理解中的区别？
**结论版：**
- **3D CNN**：卷积核在时间维上滑动，联合建模时空局部特征，训练/推理并行好，但长程依赖受限、算力开销大。
- **2D CNN + RNN/TCN**：空间与时序分离建模，时序模块能建模更长依赖（尤其 Transformer/TCN），但 RNN 串行、工程上不如 3D CNN 好并行。

**细节版：**
- 3D CNN（如 C3D/I3D/SlowFast）直接对输入 \( (B, C, T, H, W) \) 做 3D 卷积，天然捕获短期运动模式（局部时序 + 局部空间）。
- 2D CNN + 时序模块先抽帧特征 \(f_t\)，再用 RNN/TCN/Transformer 聚合，适合长视频但依赖时序模块设计与输入采样策略。

**追问反杀点：**
- 真实系统中长视频更常用“分段 + 轻量检索 + 聚合”的策略，而不是把 10 万帧直接喂模型。

---

### Q2：TimeSformer 为什么要做时空分离注意力？复杂度怎么变？
**结论版：**
- 标准 self-attention 对 \(L=T\times N\) token 复杂度 \(O(L^2)\) 很贵。
- TimeSformer 将注意力拆成**空间注意力**和**时间注意力**，在精度可接受前提下降低复杂度并更易扩展到更长序列。

**细节版：**
- 空间注意力：同一时间帧内的 patch 做 attention。
- 时间注意力：同一 patch 位置跨时间做 attention。

**追问反杀点：**
- 进一步提效可以用 sparse attention、分层结构、或先做 token 压缩（如 resampler）。

---

### Q3：VideoMAE 的核心思路是什么？为什么视频掩码率更高？
**结论版：**
- VideoMAE 是 masked autoencoder 自监督：对视频 token 做高比例掩码，预测被遮蔽部分，学习泛化表征。
- 视频时序冗余更强，因此掩码率通常比图像更高（如 90%），迫使模型学到更强的时空上下文建模能力。

**细节版：**
- Tube mask（管状掩码）让同一空间位置连续多帧被遮蔽，任务更难，鼓励模型学习真正的时序规律而非单帧捷径。

## 2. 多模态视频大模型（VLM / Video-LLM）

### Q4：Video-LLaVA / Video-ChatGPT / Video-LLaMA 这类架构的共同点是什么？
**结论版：**
- 主体都是“**视觉编码器**（图像/视频） + **对齐模块**（projector/Q-former/adapter） + **LLM**（生成/推理）”。
- 关键差异在于：视频 token 的构造与压缩、对齐方式、训练数据（指令数据质量）以及长视频处理策略。

**细节版：**
- 视觉编码器：CLIP ViT / VideoMAE / SlowFast 等。
- 对齐：MLP projector（简单稳定）、Q-Former（查询压缩）、cross-attn adapter（表达强但更贵）。
- 训练范式：对齐预训练 → 指令微调（图像指令到视频指令的迁移）。

**追问反杀点：**
- “模型能力上限”很多时候由数据决定：视频指令数据的覆盖与质量（时序推理、grounding）往往比小结构改动更关键。

---

### Q5：视觉-语言对齐（Alignment）到底在对齐什么？常见失败模式是什么？
**结论版：**
- 对齐的是：让视觉特征进入 LLM 的语义空间，使 LLM 能把视觉 token 当作“可用证据”参与 next-token 预测。
- 失败模式：**幻觉**（语言先验覆盖视觉证据）、**弱时序**（只看静态帧）、**证据漂移**（回答正确但引用错片段）。

**细节版：**
- MLP projector 过弱：可能无法表达复杂关系；过强/训练不稳：可能导致对齐崩坏，生成变得不稳定。
- 需要 grounding 约束（片段级监督、检索-生成拆分）与分桶评测才能识别“真的看视频”。

---

### Q6：两阶段训练（先对齐预训练、再指令微调）为什么有效？
**结论版：**
- 对齐预训练解决“看得懂”（表示可进入 LLM）；指令微调解决“会用”（按指令调用视觉证据、遵循格式与对话行为）。

**细节版：**
- 阶段 1：用大规模弱监督/对比学习/描述监督让视觉 token 与文本对齐。
- 阶段 2：用高质量指令数据训练模型“何时引用视觉、如何拒答、如何多轮一致”。

## 3. 视频问答（VideoQA）与跨模态推理

### Q7：设计一个视频问答系统，你会怎么做架构？
**结论版：**
- 输入：视频（分段/采样）+ 问题（tokenize）  
- 编码：视觉编码器 + 文本编码器（或直接 LLM embedding）  
- 融合：cross-attn / 多模态 transformer / 对齐模块  
- 解码：LLM 生成  
- 长视频：加检索（RAG-on-video）先召回片段，再生成

**细节版（模块拆解）：**
- **视频采样**：8/16/32 帧（短视频）；长视频用 segment + keyframe + Top-K。
- **token 压缩**：resampler / pooling，将 \(T\times N\) 压到可控长度。
- **grounding**：答案绑定到片段（时间戳或片段 id），便于评测与纠错。

**评估指标：**
- 封闭集：Accuracy；开放式：人工评估 + 生成指标（BLEU/ROUGE/CIDEr）+ grounding 指标。

---

### Q8：如何处理长视频推理效率与效果的矛盾？
**结论版：**
- 不把整条视频喂进 LLM，而是“**分段 → 召回 Top-K → 精读 + 生成**”，并配合缓存与预算控制。

**细节版：**
- 召回：用轻量视频编码器得到片段 embedding，按问题 embedding 做相似度检索。
- 精读：只对 Top-K 片段做更细粒度帧采样与 cross-attn。
- 缓存：视频片段视觉特征可缓存；同一视频多轮对话复用缓存显著降成本。

---

### Q9：你如何判断模型是真的“看了视频”而不是语言先验在猜？
**结论版：**
- 做“证据可解释 + 反事实”评测：grounding、hard negative、遮蔽对照、时序扰动、对话一致性检查。

**细节版：**
- hard negative：用相似场景片段替换证据，看答案是否跟随证据变化。
- 时序扰动：打乱片段顺序，时序问题应明显退化。
- 遮蔽：遮挡关键对象区域/关键帧，观察答案变化。

## 4. 音视频交互（Audio-Visual）

### Q10：音视频融合有哪些方式？怎么选？
**结论版：**
- 早期融合（特征级）：简单但对齐难、易噪声传播  
- 晚期融合（决策级）：稳定、工程友好  
- 交叉注意力融合：表达强但成本高、需要更好数据与正则

**如何选：**
- 看约束：线上稳定性/成本优先时偏晚期融合；追求极致效果、且有足够数据时用 cross-attn。

---

### Q11：ASR 错误会把系统带偏，怎么处理？
**结论版：**
- 置信度门控 + 纠错回流 + 噪声增强；并尽量让“视频证据”在关键问题里起主导作用。

**细节版：**
- 低置信文本不要强行作为硬条件；可并行保留音频 embedding 与视频检索结果，做多证据融合。
- 对“音频事件”类问题（爆炸/碰撞/哭声）用专门音频模型补足纯文本 ASR 的信息损失。

---

## 5. 数据挖掘、合成与治理（Data Engine）

### Q12：多模态数据链路你会如何搭建？核心质量控制点是什么？
**结论版：**
- 链路：采集 → 切分 → 抽帧/特征 → 字幕/ASR → 样本构造 → 去重过滤 → 版本管理 → 训练/评测闭环。
- 质量控制：去重、防泄漏、hard negative、噪声与合成偏差治理。

**细节版：**
- 去重：同源视频不同切片/不同分辨率也要去重，避免评测泄漏导致虚高。
- 过滤：低信息片段、低置信 ASR、合成伪影；必要时人工抽检校准过滤阈值。
- 样本结构：明确问题类型与证据片段（用于 grounding 与错误归因）。

---

### Q13：长尾场景怎么解决？为什么“只做过采样”不够？
**结论版：**
- 长尾需要“数据 + 损失 + 迁移 + 合成”组合拳；只过采样会过拟合稀有样本且无法覆盖真实多样性。

**细节版：**
- 数据：过采样 + 定向增强（遮挡、光照、运动模糊）
- 损失：类别均衡/ focal loss / re-weighting
- 迁移：大数据集预训练（Kinetics 等）→ 目标域微调
- 合成：仿真/生成式补齐稀缺事件，但必须做偏差评估与过滤

---

## 6. 评测体系（离线 + 在线）与评估系统落地

### Q14：评测维度如何拆？如何避免“离线涨、线上不涨”？
**结论版：**
- 按能力分桶（事实/时序/细粒度/鲁棒/安全/成本），并建立线上日志回流到离线评测的闭环。

**细节版：**
- 离线：每桶固定样本 + 持续扩充；同时记录召回正确率与生成正确率，做错误归因（召回错/对齐错/推理错/生成错）。
- 在线：监控拒答率、幻觉率（抽检+探针）、长尾失败原因；灰度对比看显著性而不是单点指标。

---

## 7. 推理加速与部署工程

### Q15：推理慢通常慢在哪里？你会如何系统优化？
**结论版：**
- 三大瓶颈：视频解码/预处理、视觉编码器前向、LLM 解码。必须分别优化并用 profiler 定位。

**细节版：**
- 视频侧：关键帧/分段、批量解码、I/O pipeline 优化
- 视觉侧：ONNX/TensorRT 编译、batching、特征缓存
- LLM 侧：KV cache、动态 batching、token budget、量化（INT8/FP8）、并发调度

---

### Q16：量化/编译加速怎么保证“不掉点”？你会怎么做回归？
**结论版：**
- 用校准集 + 分桶回归集，分别看“总体指标”和“关键能力桶”，并对失败样本做错误归因。

**细节版：**
- 校准集覆盖典型场景与长尾；量化后重点关注时序推理与细粒度描述是否退化。
- 回归流程固定化（模型版本、数据版本、指标口径可追溯），支持一键对比与回滚。

---

## 8. 分布式训练与训练效率

### Q17：DeepSpeed ZeRO / FSDP 分别解决什么问题？stage 怎么选？
**结论版：**
- 核心是把参数/梯度/优化器状态分片，解决显存瓶颈；stage 越高显存越省，但通信更复杂、工程更难。

**细节版：**
- ZeRO-1 分片 optimizer states；ZeRO-2 额外分片 gradients；ZeRO-3 进一步分片 parameters。
- stage 选择取决于模型规模、GPU 数量、网络带宽、可接受的吞吐损耗。

---

### Q18：你做过哪些训练提效？如何定位吞吐瓶颈？
**结论版：**
- 混合精度、梯度累积、checkpointing、通信 overlap；用 profiler 区分“算子耗时 vs 通信耗时 vs dataloader”。

**细节版：**
- dataloader 常见瓶颈：视频解码慢、随机访问多；解决：预解码、缓存、并行 worker、异步 pipeline。
- 通信瓶颈：bucket/融合策略、梯度同步频率、all-reduce overlap。

## 9. 项目深挖通用答题模板（STAR + 技术细节清单）

### Q19：请介绍一个你做过的视频理解/多模态项目（建议必背）
**回答骨架（STAR）：**
- **S**：业务背景、场景约束（时延/吞吐/成本/线上质量）
- **T**：要解决的核心问题（长视频、对齐、时序推理、长尾）
- **A**：方案（模型/数据/评测/工程）+ 关键细节（token 预算、对齐方式、检索-生成、量化回归）
- **R**：指标提升（效果+成本）+ 上线影响 + 复用沉淀

**技术细节清单（被追问时逐条展开）：**
- 输入：帧采样策略、token 长度预算、长视频分段方式
- 模型：视觉编码器选择、对齐模块选择、训练阶段与冻结策略
- 推理：grounding、检索-生成、缓存、并发调度
- 数据：去重过滤、hard negative、合成与偏差评估
- 评测：分桶评测、错误归因、线上回流闭环
- 加速：编译/量化、精度回归、灰度发布
